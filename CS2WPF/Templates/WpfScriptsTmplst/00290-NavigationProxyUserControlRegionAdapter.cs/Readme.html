Prism application Class Library
Generator group Context level

Remark

Using "prism:RegionManager.RegionName" does not provide the ability to apply XAML-Binding at design time.
Using 00290-NavigationProxyUserControlRegionAdapter.cs and 00288-NavigationProxyUserControl.xaml.cs(00288-NavigationProxyUserControl.xaml) makes this feature to be avalable.
In the App.xaml.cs-file of the Prism application the following code should be added:
        
            protected override void ConfigureRegionAdapterMappings(RegionAdapterMappings regionAdapterMappings)
            {
                base.ConfigureRegionAdapterMappings(regionAdapterMappings);
                regionAdapterMappings.RegisterMapping < NavigationProxyUserControl, NavigationProxyUserControlRegionAdapter > ();
            }
