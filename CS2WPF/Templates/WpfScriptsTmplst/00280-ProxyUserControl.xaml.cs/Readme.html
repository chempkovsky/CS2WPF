Remark

Using "prism:RegionManager.RegionName" does not provide the ability to apply XAML-Binding at design time.
Using 00284-ProxyUserControlRegionAdapter.cs and 00280-ProxyUserControl.xaml.cs(00282-ProxyUserControl.xaml) makes this feature to be avalable.
In the App.xaml.cs-file of the Prism application the following code should be added:
        
            protected override void ConfigureRegionAdapterMappings(RegionAdapterMappings regionAdapterMappings)
            {
                base.ConfigureRegionAdapterMappings(regionAdapterMappings);
                regionAdapterMappings.RegisterMapping < ProxyUserControl, ProxyUserControlRegionAdapter > ();
            }
